TASK 01 — PRIORITY FRAMEWORK

Frequency × Impact 매트릭스

단순 빈도 순위는 "가장 많이 언급된 것"을 올리고, 단순 심각도 순위는 "가장 급한 것"만 올립니다. 두 축을 곱했을 때만 "지금 당장 팀이 집중해야 할 곳"이 교차점으로 드러납니다.

📊

구독 모델의 LTV 특성

SaaS에서 한 기업고객의 이탈은 단건 매출이 아닌 수년치 ARR의 손실입니다. 빈도가 낮아도 기업고객 이탈 신호는 임팩트가 기하급수적이므로, 두 축의 곱이 올바른 가중치를 자동으로 부여합니다.

🔁

반복 버그의 누적 피해

작업·문서·결제는 매일 접하는 핵심 워크플로우입니다. 1회성 불편이 아닌 반복 발생하는 버그는 빈도가 그 자체로 심각도 신호이며, Frequency 축이 이 누적 피해를 수치화합니다.

⚖️

"가장 큰 목소리" 편향 제거

CS 티켓은 불만을 잘 표현하는 고객이 과대 대표됩니다. Impact 점수에 user_type 가중치를 두면 조용히 이탈하는 기업고객의 신호도 정량화해 편향을 상쇄합니다.

임팩트 점수 산정 방식

4가지 차원을 곱해 행 단위 임팩트를 계산합니다. 이탈 위험 신호(해지·분쟁·탈퇴)는 매출 즉각 손실 위협이므로 ×2 하드 가중치를 별도 적용합니다.

Priority Score = Frequency × ( U × S × E × C ) × ChurnMultiplier
U = 고객등급 · S = 심각도 · E = 감정 · C = 유형 · ChurnMul = 2× (이탈위험 감지 시)

함수별 설계 명세

load_data()
filepath: str → pd.DataFrame
CSV 로드 + 날짜 컬럼 파싱. 매주 새 파일 경로만 바꾸면 됩니다. 누적 파일도 지원합니다.
clean_data()
df: DataFrame → DataFrame
id 기준 중복 제거, 결측값 기본값 보충, 공백 제거. 채널·user_type 오타 정규화 포인트.
enrich_classification()
df: DataFrame → DataFrame
is_churn_risk, is_payment_issue, is_paid_customer 파생 컬럼 추가. 오분류 버그 재라벨링 안전망 포함.
apply_priority_matrix()
df: DataFrame → DataFrame
행 단위 impact_score 계산 후 유형분류 단위 frequency 집계. priority_score = frequency × impact.
section_cs_top5()
df: DataFrame → list[dict]
심각도 높음/부정 감정/이탈 위험 필터 후 impact_score 상위 5건 반환.
section_dev_top3()
df: DataFrame → list[dict]
버그 유형 필터 후 frequency → impact_score 순 정렬, 상위 3건 반환.
section_pm_features()
df: DataFrame → list[dict]
기능요청 + 유료/기업 고객 필터 후 priority_score 상위 5건 반환.
assemble_report()
(df, date, cs, dev, pm) → str
세 섹션 + KPI 스냅샷을 Markdown으로 조립. Notion·Confluence·Slack 붙여넣기 호환.

메인 진입점 — run_pipeline()

voc_pipeline.py Python 
def run_pipeline(
    input_filepath: str,
    output_dir: str = "./reports",
    report_date: str | None = None,
) -> Path:
    # 날짜 기본값: 오늘
    if report_date is None:
        report_date = datetime.today().strftime("%Y-%m-%d")

    df = load_data(input_filepath)      # 1. 로드
    df = clean_data(df)                # 2. 정제
    df = enrich_classification(df)     # 3. 보강
    df = apply_priority_matrix(df)     # 4. 점수화

    cs_items  = section_cs_top5(df)   # 5a. CS 섹션
    dev_items = section_dev_top3(df)   # 5b. 개발 섹션
    pm_items  = section_pm_features(df) # 5c. 기획 섹션

    report = assemble_report(          # 6. 조립
        df, report_date, cs_items, dev_items, pm_items
    )
    return save_report(report, output_dir, report_date) # 7. 저장


# 실행: python voc_pipeline.py cleaned_voc.csv ./reports
if __name__ == "__main__":
    import sys
    run_pipeline(sys.argv[1], sys.argv[2] if len(sys.argv) > 2 else "./reports")
